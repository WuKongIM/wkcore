# test/e2e AGENTS

本文件仅约束 `test/e2e` 目录。

- 保持进程级黑盒 e2e：通过真实进程、真实协议、真实公开入口验证行为，不要导入 `internal/app`。
- 优先验证外部可观察结果，不要依赖内部状态、内部 hook 或测试专用捷径。
- 不要使用固定等待时间；优先使用健康检查、握手探测、轮询和最终一致判断。
- 通用测试辅助能力放在 `test/e2e/suite`，避免把业务判断散落到各个测试文件。
- 失败诊断保持有界：优先输出必要的配置、stdout/stderr、节点日志尾部和关键管理面响应。
- 当 e2e 测试需要额外观测数据时，优先复用现有 `/manager/` 相关 API；如果现有接口不足并需要新增 `/manager/` 相关 API，必须先询问开发者并获得确认，再开始编写。
